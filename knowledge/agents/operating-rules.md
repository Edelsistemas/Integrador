# Operating Rules for AI-Assisted Development

1. No modificar código sin leer la spec activa.
2. No implementar specs en estado Draft salvo autorización explícita.
3. Usar contexto mínimo necesario.
4. No inventar arquitectura.
5. No hardcodear secretos.
6. No declarar éxito sin harness.
7. Actualizar documentación relevante.
8. Registrar desvíos y decisiones.
9. Requerir revisión humana para cambios sensibles.
10. No activar batches contra producción desde entorno local.
