# Commit policy

- Ningun commit sin `HUMAN_GATE: AUTHORIZE_COMMIT`.
- Ningun push sin autorizacion explicita.
- Separar documentacion de codigo cuando convenga.
- Revisar `git diff` y `git status` antes de commitear.
- No incluir secretos, temporales, datos persistidos ni cambios fuera de scope.
- Si el repo esta sucio, definir plan de staging por grupos antes de commitear.
- Durante bootstrap Wiki-SDD no hacer commit ni push en esta ejecucion.
